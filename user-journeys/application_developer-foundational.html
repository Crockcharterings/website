If you're an application developer looking to get started with Kubernetes, this page and its linked topics introduce you to foundational concepts and tasks. The focus is on a development workflow -- check out other user journeys to learn about managing production clusters, or providing high availability (HA) for critical application deployments.

What can Kubernetes do for me? Kubernetes is commonly described as an orchestration platform for containerized applications. Containers provide lightweight portable packaging for applications, and Kubernetes helps deploy your containers to a cluster of connected computers and then manage the cluster. The cluster can run on bare metal, in classic virtual machines, or in the cloud. For a fuller description of Kubernetes and more information about containers, see What Is Kubernetes. For more information about different environments for deploying Kubernetes, see LINK.

One good way to learn Kubernetes basics is with Minikube, which lets you create a simple single-node cluster. You can explore Minikube with the interactive online tutorials, or you can install it locally. If you install locally, you may find that your cluster is sufficient for local application development. Installing Minikube locally is also a good idea if you want to continue exploring in a persistent environment that you can come back to and change.

Using Minikube to Create a Cluster

If you're brand new to Kubernetes, you might want to try out the Katacoda Kubernetes Playground, which provides a set of web-based tutorials complete with their own Kubernetes environment.

If you want to explore further, the web-based Play with Kubernetes gives you a less structured Kubernetes environment on the web.



1  Set up Kubernetes
To set up and explore your own Kubernetes cluster for the first time, you can create a simple cluster locally with Minikube. Minikube gives you a cluster with a single node and all core Kubernetes components. You can set up a Minikube cluster just to explore, or, depending on your application, Minikube can also provide a useful development environment.

Before you install Minikube:
  • Install a hypervisor if you don't have one already:
  • For OS X, xhyve, VirtualBox, or VMWare Fusion
  • For Linux, VirtualBox or KVM
  • For Windows, VirtualBox or Hyper-V
  • Install Docker. Minikube includes a Docker daemon, but we recommend that you install your own. Note that version 1.12 is recommended, but a few other versions are tested and known to work.
  • Install kubectl, the Kubernetes command-line tool. Note that when you start Minikube, your kubectl is automatically configured to communicate with the server. You should therefore skip kubectl configuration at this point.

3  Understand Kubernetes basics
Kubernetes is an open source system for managing containerized applications across multiple hosts, or nodes. It provides basic mechanisms to deploy, maintain, and scale your application.

To work with Kubernetes, you describe your cluster's desired state in terms of Kubernetes API objects. Cluster state includes but is not limited to the following information:
  • The applications or other workloads to run
  • The container images for your applications and workloads
  • Allocation of network and disk resources


Once you’ve set your desired state, the Kubernetes Control Plane works to make the cluster’s current state match the desired state. To do so, Kubernetes performs a variety of tasks automatically–such as starting or restarting containers, scaling the number of replicas of a given application, and more. The Kubernetes Control Plane consists of a collection of processes running on your cluster:
  • The Kubernetes Master is a collection of three processes that run on a single node in your cluster, which is designated as the master node. Those processes are: kube-apiserver, kube-controller-manager and kube-scheduler.
  • Each individual non-master node in your cluster runs two processes:
    • kubelet, which communicates with the Kubernetes Master.
    • kube-proxy, a network proxy which reflects Kubernetes networking services on each node.
